# 数字化转型办公室 - 文件自动整理 Agent

> **版本：v5.0** | 支持 Web 界面 + CLI 双模式

自动化文件整理工具，根据文件名关键词和业务规则，将散落的文件分类归档到标准目录结构中。

---

## 功能特性

| 特性 | 说明 |
|------|------|
| 三级优先级匹配 | 业务领域 > 关键词 > 扩展名兜底 |
| 归档保留路径 | 归档文件时保留原始相对路径结构 |
| 规则命中率统计 | 每次整理输出规则命中率，便于规则优化 |
| Web 管理界面 | 浏览器可视化操作，支持任务管理 |
| CLI 完整参数 | 支持脚本化、自动化、定时任务 |
| 回滚机制 | 每次整理自动备份，支持一键回滚 |
| 守护模式 | `--watch` 持续监控，自动化执行 |
| 重复文件检测 | 检测并处理同名/同内容文件 |

## 快速开始

### 1. 安装依赖

```bash
pip install flask flask-cors
```

### 2. 配置源目录

编辑 `config.json`，修改 `source_dir` 为你的文件目录：

```json
{
  "source_dir": "C:\\数字化办公室SVN",
  "target_dir": ""   // 空=同目录内归档，也可指定其他目录
}
```

### 3. 预览整理效果（推荐先预览）

```bash
python run.py --preview
```

### 4. 执行实际整理

```bash
python run.py --execute
```

## CLI 完整参数速查

### 模式开关（互斥，选其一）

| 参数 | 说明 |
|------|------|
| `--preview` | 预览模式（默认，不移动文件） |
| `--execute` | 执行整理（实际移动文件） |
| `--watch` | 守护模式：持续监控文件变化并整理 |
| `--rollback` | 回滚上一次整理操作 |
| `--stats` | 查看历史整理统计 |
| `--web` | 启动 Web 管理界面 |
| `--setup` | 初始化标准目录结构（不移动文件） |

### 配置选项

| 参数 | 说明 | 示例 |
|------|------|------|
| `--source <path>` | 指定源目录（覆盖配置文件） | `--source D:\我的文件` |
| `--target <path>` | 指定目标目录（覆盖配置文件） | `--target D:\归档目录` |
| `--template <name>` | 指定规则模板名称 | `--template digital-office` |
| `--rules <json>` | 直接传入 JSON 规则覆盖 | 见下方"指定规则"示例 |
| `--keep-original` | 复制模式（不移动，保留原文件） | `--execute --keep-original` |
| `--config <path>` | 指定配置文件路径 | `--config my-config.json` |

### 扫描控制

| 参数 | 说明 | 示例 |
|------|------|------|
| `--days <N>` | 仅扫描最近 N 天修改的文件 | `--days 3` |
| `--full` | 全量扫描（不限修改时间） | `--execute --full` |
| `--duplicates` | 检测重复文件 | `--duplicates --execute` |

### Web 界面选项

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--port <N>` | Web 服务端口 | `5000` |
| `--host <addr>` | Web 监听地址 | `0.0.0.0` |

## 使用示例

### 基础用法

```bash
# 预览整理效果（推荐先用预览）
python run.py --preview

# 执行整理（移动文件）
python run.py --execute

# 整理最近 7 天内修改的文件
python run.py --execute --days 7

# 全量整理（所有文件，不限时间）
python run.py --execute --full
```

### 源目录与目标目录分离

```bash
# 源和目标分开（源不变，目标另存归档）
python run.py --execute --source "C:\源文件夹" --target "D:\归档文件夹"

# 只指定源（默认在源目录内归档）
python run.py --execute --source "C:\源文件夹"

# 只指定目标
python run.py --execute --target "D:\归档文件夹"
```

### 指定分类规则

通过 `--rules` 参数直接在 CLI 传入 JSON 规则，无需修改模板文件：

```bash
# 添加一条关键词规则（所有含"合同"的文件归入合同目录）
python run.py --execute --rules "{\"keyword_rules\":[{\"keywords\":[\"合同\",\"协议\"],\"target\":\"07-商务与采购\\00-合同协议\"}]}"

# 组合多个规则
python run.py --execute --rules "{\"keyword_rules\":[{\"keywords\":[\"合同\"],\"target\":\"07-商务与采购\\00-合同协议\"},{\"keywords\":[\"发票\"],\"target\":\"08-财务文档\\01-票据报销\"}]}"
```

### 复制模式（保留原文件）

```bash
# 执行整理但保留原文件（复制到目标，原文件不动）
python run.py --execute --keep-original
```

### 初始化目录结构

```bash
# 预览：将创建哪些目录
python run.py --setup

# 执行：实际创建目录
python run.py --setup --execute
```

### 回滚操作

```bash
# 回滚上一次整理（将文件移回原位置）
python run.py --rollback
```

### 重复文件处理

```bash
# 预览重复文件
python run.py --duplicates

# 执行处理（移动重复文件）
python run.py --duplicates --execute
```

### 守护模式（自动化）

```bash
# 每 5 分钟自动整理一次（间隔可在 config.json 中配置 watch_interval_seconds）
python run.py --watch

# 指定源和目标后启动守护模式
python run.py --watch --source "C:\数字化办公室SVN" --target ""
```

### Web 管理界面

```bash
# 默认端口 5000
python run.py --web

# 指定端口
python run.py --web --port 8080

# 指定监听地址（仅本机访问）
python run.py --web --host 127.0.0.1 --port 5000
```

然后浏览器打开 `http://localhost:5000`（或你指定的端口）。

## Web 界面使用

1. **创建任务**：输入源目录、目标目录，选择模板，点击"启动整理"
2. **查看状态**：任务执行中实时显示扫描进度和运行时长
3. **查看日志**：点击日志文件名查看完整转移记录（包含转移路径和匹配规则）
4. **规则命中率**：在日志中可查看各规则的命中次数，用于优化规则
5. **编辑任务**：修改源/目标目录和模板参数

> **注意**：Web 界面执行任务为异步后台运行，关闭浏览器不影响正在执行的任务。

## 分类规则说明

### 三级优先级

1. **优先级 1 - 业务领域匹配**：根据文件名中是否包含特定业务关键词，直接匹配项目目录
2. **优先级 2 - 关键词规则**：按关键词分类到对应目录（v5.0 扩展至 32+ 条规则）
3. **优先级 3 - 扩展名兜底**：按文件类型（Word/Excel/PPT/图片等）归入对应扩展名目录

## 日志与规则优化

### 查看整理日志

每次整理后，日志保存在 `logs/session_YYYYMMDD_HHMMSS.json`，包含：
- 每条文件的转移记录（源路径 → 目标路径）
- 匹配的规则名称和优先级
- 规则命中率统计（rule_hit_stats）
- 未匹配文件列表

### 规则命中率统计

每次整理后在控制台输出规则命中率，例如：

```
📊 规则命中率统计（共 8 条规则被命中）:
  120 次  00-战略与架构\00-数字化战略
   45 次  01-知识库管理\02-培训学习
   23 次  07-商务与采购\00-合同协议
   ...
```

**优化方向**：命中率高的规则说明匹配精准；命中率低或为 0 的规则可以考虑删除或调整关键词。

### 日志文件字段说明

session JSON 日志中的 `entries` 字段每条记录包含：

| 字段 | 说明 |
|------|------|
| `source` | 源文件完整路径 |
| `target` | 目标文件完整路径 |
| `source_rel` | 源文件相对于源目录的路径 |
| `target_rel` | 目标文件相对于目标目录的路径 |
| `rule` | 匹配到的规则名称（描述性文本） |
| `priority` | 匹配优先级 1/2/3 |
| `status` | 操作状态 success/skip/error |
| `timestamp` | 操作时间 |

## 常见问题

### Q: 不想移动文件，只想预览会整理到哪些位置？

```bash
python run.py --preview
```

### Q: 想把不同目录的文件整理到同一个归档目录？

使用 `--source` 指定不同目录，`--target` 指定同一目标目录即可：
```bash
python run.py --execute --source "C:\目录A" --target "D:\统一归档"
```

### Q: 想临时添加一条分类规则，不想改模板文件？

使用 `--rules` 参数：
```bash
python run.py --execute --rules "{\"keyword_rules\":[{\"keywords\":[\"年报\"],\"target\":\"07-商务与采购\\00-合同协议\"}]}"
```

### Q: 整理后发现有问题，想撤回？

```bash
python run.py --rollback
```
回滚后文件会移回整理前的原位置。

### Q: 想定期自动整理，不用手动运行？

```bash
python run.py --watch --source "C:\数字化办公室SVN"
```
守护模式会按 `config.json` 中的 `watch_interval_seconds`（默认 300 秒）持续监控并整理。

### Q: 太多文件进了"归档"目录，怎么减少？

查看日志中的规则命中率，将高频误归档的扩展名兜底文件对应的关键词添加到 `keyword_rules` 中，减少走优先级 3兜底 的文件数量。

### Q: Web 界面任务执行时间很长，会超时吗？

v5.0 已优化：前端超时增大到 120 秒，轮询间隔增大到 3 秒，任务运行时长实时显示在界面。如果仍需处理大量文件，建议使用 CLI 的 `--watch` 模式后台运行。

## 配置文件说明

`config.json` 主要配置项：

| 配置项 | 说明 | 默认值 |
|-------|------|--------|
| `source_dir` | 源文件目录 | - |
| `target_dir` | 目标目录（空=同目录内归档） | 空 |
| `dry_run` | 默认预览模式 | `true` |
| `keep_original` | 默认复制模式（不移动） | `false` |
| `scan_days` | 默认扫描天数 | `7` |
| `watch_interval_seconds` | 守护模式轮询间隔 | `300` |
| `large_file_threshold_mb` | 大文件阈值（MB） | `100` |
| `protected_folders` | 跳过不扫描的目录 | SVN/Git 等 |
| `delete_patterns` | 自动删除的临时文件模式 | ~$开头的临时文件等 |

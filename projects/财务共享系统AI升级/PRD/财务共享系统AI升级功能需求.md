## 3. 需求详情

### 3.1 CH-GN260501：发票拆票推送百望优化

**优先级**：P1

**需求提出人**：关键用户

**功能说明**：
LINK2ACC_STATUS_VAT_BATCH 百望发票信息推送接口：拆票场景下的状态推送优化。

**解决方案**：
1. **现状**：拆票目前推送已审核状态 `accountStatus = -1`
2. **改造逻辑**：
   - 单据拆票完成报销时，汇总所有已报销子票的实际报销税额
   - 如果汇总税额等于母票税额，则推送状态传 `accountStatus = -2`（已记账）
   - 如果汇总税额不等于母票税额，则推送状态传 `accountStatus = -1`（已审核）
3. **推送金额逻辑不变**：还是当次实际报销税额

**接口规范**：
```
接口：LINK2ACC_STATUS_VAT_BATCH
字段：accountStatus
  - -1：已审核
  - -2：已记账
条件：拆票完成报销时，判断子票税额汇总是否等于母票税额
```

### 3.2 CH-GN260502：共享单据相关预警落表

**优先级**：P0

**需求提出人**：关键用户

**功能说明**：
后台增加 AI 预警表，单据每次提交成功时，如有相关规则提示，则写入预警表。

**数据模型——AI预警表**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| submitter_no | TEXT | 提单人工号 |
| bill_no | TEXT | 单据编号 |
| warning_message | TEXT | 预警提示信息 |
| created_at | TEXT | 创建时间 |

**AI预警规则（8条）**：
1. 借款核销提醒
2. 预付核销提醒
3. 住宿超标提醒
4. 预提关联提醒
5. 供应商黑名单提醒
6. 发票购买方和核算主体校验
7. 发票销售方和供应商校验
8. 发票不合规提醒

**解决方案**：
1. **后台写入**：单据每次提交成功时，如有相关规则提示，则写入预警表
2. **界面展示**：在初审审核界面 - 智能审单区域最下方增加预警信息展示
3. **展示规则**：展示最后一次单据提交的预警情况，每个预警规则显示一条数据
4. **AI推送标识**：如果有任何一项预警信息，给AI推送审核信息时推送 `warning_flag = "Y"`，没有则推送 `warning_flag = "N"`

**界面说明**：
- 共享提醒规则结果区域显示预警信息列表
- 每条预警规则显示一条数据

### 3.3 CH-GN260503：合同接口部分功能调整

**优先级**：P1

**需求提出人**：关键用户

**功能说明**：
合同同步和列表接口进行调整，涉及字段映射和查询条件变更。

**解决方案**：

1. **合同同步接口字段调整**：
   - 不能直接取客商编码
   - 应取接口字段 `supplierEBSID`，对应 `zfs_sys.sys_vendors` 表的 `EXTEND_S1` 字段

2. **合同列表同步接口**：
   - 使用 `pubtsBegin` 字段同步（不用 `valiDateBegin`）

3. **合同列表请求接口查询条件**：
   - `bustype: "CWGX"`
   - 新增值：`EBSJCWGX`
   - 固定推送：`bustype: "CWGX,EBSJCWGX"`

### 3.4 CH-GN260504：审批授权增加单据类型规则

**优先级**：P1

**需求提出人**：关键用户

**功能说明**：
审批授权规则增加单据类型维度，实现更细粒度的授权控制。

**解决方案**：
1. 在审批授权规则中增加"单据类型"多选框
2. 空代表全部（不限制单据类型）
3. 勾选代表指定单据类型（多选）

---

### 3.5 CH-GN260505：增加AI问答入口

**优先级**：P0

**需求提出人**：关键用户

**功能说明**：
在系统所有界面增加 AI 问答入口，用户可通过图标点击打开财务机器人会话。

**解决方案**：
1. **全局图标**：在系统所有界面增加图标（AI问答入口）
2. **闪信接口调用**：
   - 点击图标后请求闪信接口
   - 检查闪信有没有打开（端口检测）
   - 没打开则报错，提示用户自行打开，不做程序拉起
3. **打开会话**：闪信正常后，调用另一个接口打开和财务机器人的会话

---

### 3.6 CH-GN260506：批量上传发票改造

**优先级**：P1

**需求提出人**：关键用户

**功能说明**：
PC 端电子发票上传功能增强，支持多文件选择和错误提示。

**解决方案**：
1. **上传按钮改造**：电子发票上传按钮改为可多选发票文件
2. **数量限制**：最多可选择 20 张发票
3. **错误处理**：
   - 如果失败，在界面报错：文件名称 + 错误信息
   - 报错信息不自动关闭
   - 用户点 X 或关闭按钮关闭报错
4. **重名处理**：WEB 端也做重名处理

### 3.7 CH-GN260507：新增业务类型和发票明细的关系菜单

**优先级**：P1

**需求提出人**：关键用户

**功能说明**：
在基础数据平台增加业务类型与发票明细的关系维护功能。

**解决方案**：
1. **菜单位置**：基础数据平台 → 报账基础 → 业务类型与发票明细
2. **查询条件**：
   - 业务大类：LOV 选择框，单选
   - 业务小类：LOV 选择框，单选
   - 发票项目：文本，精确查询
   - 是否有效：下拉框
3. **按钮**：新增、删除、启用、禁用、下载模板、导入模板、导出

**新增界面字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 业务大类 | LOV选择框 | 是 | 单选 |
| 业务小类 | LOV选择框 | 否 | 单选 |
| 发票项目名称 | 文本 | 是 | 手工输入 |

**结果列表字段**：业务大类、业务小类、发票项目、是否有效


### 2. 文件分类太粗暴问题 ✅ 已增强

**原有问题：**
- 仅根据文件名关键词匹配
- 无法识别文件内容语义
- 分类粒度较粗

**v4.0 增强功能：**

#### 2.1 四级优先级分类体系
1. **P1 业务领域匹配**（最高优先级）- 文件名匹配业务关键词
2. **P2 关键词规则匹配** - 文件名匹配通用关键词
3. **P3 文件内容智能分析**（新增）- 读取文档内容进行语义分析
4. **P4 扩展名兜底分类** - 按文件类型兜底

#### 2.2 内容智能分析功能
- **支持格式**：Word(.docx)、PDF、Excel(.xlsx)、PPT(.pptx)、文本文件等
- **分析方法**：提取文档内容，匹配业务关键词库
- **置信度评分**：根据关键词匹配程度给出置信度

#### 2.3 文件名智能分析
- 自动识别日期格式（2024-01-15、20240115等）
- 自动识别项目编号（ERP-001、PRJ001等）
- 自动识别版本号（V1.0、v2.5等）
- 自动识别文档类型（需求、设计、测试、报告等）

#### 2.4 新增依赖包
```
python-docx>=0.8.11    # Word文档读取
PyPDF2>=3.0.0          # PDF内容提取
openpyxl>=3.0.0        # Excel内容读取
python-pptx>=0.6.21    # PPT内容提取
striprtf>=0.0.26       # RTF格式支持
```

## 使用方法

### 安装依赖
```bash
pip install -r requirements.txt
```

### 启动 Web 界面
```bash
python run.py --web
```

### 命令行使用
```bash
# 预览模式（默认）
python run.py --preview

# 执行整理
python run.py --execute

# 启用内容分析（默认已启用）
python run.py --execute
```

### 配置内容分析
在规则模板中配置 `content_analysis_rules`：
```json
{
  "content_analysis_rules": {
    "enabled": true,
    "max_file_size_mb": 10,
    "supported_extensions": [".txt", ".md", ".docx", ".pdf", ...],
    "content_keywords": [
      {"keywords": ["客舱", "cabin"], "target": "03-数字化项目\\00-客舱数字化", "weight": 3},
      ...
    ]
  }
}
```

- 内容分析支持中文关键词匹配

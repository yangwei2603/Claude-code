# 量化策略使用说明

> 波动率目标仓位量化回测框架，用于研究与模拟盘验证

## 警告

⚠️ 该脚本用于**研究与回测**，不应直接当作实盘自动交易系统。

## 快速运行

```bash
python3 tools/fx_quant_strategy.py \
  --csv reports/sample_usdcny.csv \
  --out reports/fx_signals_sample.csv \
  --lookback 20 \
  --vol-window 30 \
  --target-vol 0.10 \
  --max-leverage 2.0 \
  --stop-loss 0.02
```

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--csv` | 输入 CSV 路径（字段 `timestamp, close`，可选 `spread_bps`） | 必填 |
| `--out` | 输出信号 CSV 路径 | 必填 |
| `--lookback` | 趋势计算回望期（天） | 20 |
| `--vol-window` | 波动率计算窗口（天） | 30 |
| `--target-vol` | 目标年化波动率 | 0.10（10%） |
| `--max-leverage` | 最大杠杆上限 | 2.0 |
| `--stop-loss` | 单期止损阈值（净收益下限） | 0.02 |

## 核心风控机制

### 1. 波动率目标仓位

使用滚动波动率缩放仓位，避免高波动期间过度暴露：

```
仓位 = (目标波动率 / 实际滚动波动率) * 方向信号
```

### 2. 杠杆上限

`--max-leverage` 硬限制绝对仓位，防止极端行情爆仓。

### 3. 交易成本惩罚

按仓位变化（turnover）乘以点差（bps）扣减收益：

```
成本 = |新仓位 - 旧仓位| × spread_bps
```

### 4. 单期止损封顶

`--stop-loss` 对单期净收益做下限保护，触发后本期仓位归零。

## 建议上线流程

1. **数据准备**：接入真实历史数据（至少 3-5 年，含不同宏观周期）
2. **回测验证**：滚动窗口 walk-forward 验证，避免参数过拟合
3. **压力测试**：交易时段、节假日、极端行情（2020 疫情、2022 美联储加息等）
4. **模拟盘**：运行 2-4 周，验证成交滑点与实盘一致性
5. **小资金试运行**：达到风险阈值后，小资金试运行
6. **实盘接入**：全流程风控就位后再考虑

## 输出说明

输出 CSV 包含字段：

- `timestamp`：日期
- `close`：收盘价
- `signal`：仓位信号（-1 到 1）
- `volatility`：滚动波动率
- `leverage`：实际杠杆
- `pnl`：单期盈亏
- `cum_pnl`：累计盈亏
- `cost`：交易成本

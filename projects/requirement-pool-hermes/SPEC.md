### 1.6 动效规范

**过渡时长**

| 类型 | 时长 | CSS 变量名 |
|------|------|-----------|
| 快速悬停反馈 | 150ms | `--transition-fast` |
| 标准状态切换 | 250ms | `--transition-normal` |
| 展开/收起 | 300ms | `--transition-slow` |

**缓动函数**

```css
--ease-out: cubic-bezier(0.4, 0, 0.2, 1);        /* 标准退出 */
--ease-in-out: cubic-bezier(0.4, 0, 0.2, 1);     /* 标准往返 */
--ease-spring: cubic-bezier(0.175, 0.885, 0.32, 1.275); /* 弹性效果 */
```

**动效场景**

| 场景 | 动效描述 |
|------|---------|
| 按钮悬停 | 背景色加深（150ms ease-out） |
| 卡片悬停 | 上移2px + 阴影加深（250ms ease-out） |
| 下钻点击 | 卡片缩放至0.98后跳转（150ms） |
| 模态框出现 | 背景遮罩淡入 + 弹窗从顶部滑入（300ms ease-out） |
| 模态框关闭 | 背景遮罩淡出 + 弹窗向上滑出（250ms ease-in） |
| Tab 切换 | 内容淡入切换（200ms） |
| 下拉展开 | 高度从0展开 + 淡入（200ms ease-out） |
| 加载状态 | 骨架屏闪烁动画（1.5s infinite） |
| 警告提示 | 从右侧滑入（300ms ease-spring） |
| Toast 通知 | 从顶部滑入 + 3秒后自动消失 |

## 3. 功能详情

### 3.1 首页看板

**KPI 指标卡（5个）**

| 指标 | 计算逻辑 | 颜色 |
|------|---------|------|
| 总需求数 | `SELECT COUNT(*) FROM requirements` | 主色蓝 |
| 进行中 | `SELECT COUNT(*) FROM requirements WHERE status = 'active'` | 主色蓝 |
| 已关闭 | `SELECT COUNT(*) FROM requirements WHERE status = 'closed'` | 灰色 |
| 待评审 | `SELECT COUNT(*) FROM requirements WHERE is_reviewed = 0` | 橙色 |
| 本月新增 | `SELECT COUNT(*) FROM requirements WHERE created_at >= 本月1号` | 主色蓝 |

- 指标卡样式：白底卡片，数字大号加粗，标签小号灰色
- 悬停：卡片上浮 + 阴影加深
- 点击：跳转至对应筛选视图的需求列表页

**积压超时预警卡片**

- 触发条件：需求导入后超过 20 个工作日未评审（排除周末）
- 样式：白色卡片，左侧红色 4px 边框，红色警告图标
- 内容：超时需求列表（需求编号、需求名称、超期天数、最后修改时间）
- 超期天数计算：`(当前日期 - business_created_date) - 周末天数 - 20`
- 列表最多显示 10 条，超出显示"查看全部"链接
- 点击任意需求：跳转至评审管理页面，自动带上该需求的筛选条件

**无归属项目需求提醒**

- 触发条件：评审通过但 `project_id` 为空的记录
- 样式：白色卡片，左侧橙色 4px 边框，橙色提示图标
- 内容：未关联项目的需求列表（需求编号、需求名称、通过日期）
- 操作：每条需求显示"快速立项"按钮

**多维度分布图（CSS 实现，无图表库）**

| 分布类型 | 实现方式 | 点击行为 |
|---------|---------|---------|
| 评审质量分布 | 饼图（CSS conic-gradient） | 下钻到已评审列表 |
| 状态分布 | 横向条形图 | 下钻到对应状态列表 |
| 创建人分布 | 横向条形图 + 数量标签 | 下钻到该创建人全部需求 |
| 部门分布 | 横向条形图 + 数量标签 | 下钻到该部门全部需求 |
| 进行中项目 | 表格形式 | 下钻到该项目关联需求 |

**当月评审统计**

- 本月评审通过数量（绿色）
- 本月评审驳回数量（红色）
- 本月规划数量（灰色）

### 3.2 CSV 导入

**模板下载**

- 按钮文字："下载导入模板"
- 下载文件：`demand_import_template.csv`
- 说明文字：包含必填字段说明和示例行

**文件上传**

- 支持拖拽上传（拖拽区域虚线边框，悬停时边框变实心蓝色）
- 支持点击上传（触发文件选择框）
- 仅支持 `.csv` 格式
- 文件大小限制：10MB
- 上传前校验：文件格式、必填字段是否完整

**导入结果展示**

| 结果类型 | 数量 | 说明 |
|---------|------|------|
| 新增待评审 | N | 无历史记录，新建评审记录 |
| 已有未变化 | N | hash 未变，保持现有状态 |
| 已有已更新 | N | hash 变化，标记更新，重新待评审 |
| 导入失败 | N | 格式错误或数据校验失败 |

- 结果列表以表格形式展示详情
- 状态分布以横向条形图可视化

### 3.3 评审管理

**Tab 1：待评审**

- 筛选器：创建人（多选）、部门（多选）、创建时间范围（日期区间）、优先级（多选）
- 列表字段：勾选框、需求编号、需求名称、部门、创建人、创建时间、优先级、操作
- 操作：单个"评审"按钮，或批量"评审选中"
- 批量评审：选中多条后弹窗依次录入（仅支持同类评审结果批量处理）

**Tab 2：已评审（本次）**

- 列表字段：需求编号、需求名称、评审分类、优先级、项目、操作
- 操作：
  - "确认录入禅道"按钮 → 标记 `is_confirmed=1`，移动到历史Tab
  - "编辑"按钮 → 重新打开评审弹窗
  - "撤回"按钮 → 恢复为待评审状态

**Tab 3：历史评审**

- 列表字段：需求编号、需求名称、评审分类、确认时间、禅道录入状态
- 操作：仅查看，不可编辑
- 支持按确认时间范围筛选

**评审弹窗交互**

1. 弹窗从顶部滑入，背景遮罩半透明黑色
2. 选择"评审分类"后，根据选择动态显示/隐藏条件字段
3. 通过时：优先级（必选）、计划上线时间（必选，日期选择器）、项目归属（必选）
4. 项目归属选择"新建项目"：显示项目名称 + 负责人输入框
5. 项目归属选择"归属现有项目"：显示项目下拉列表
6. 驳回时：驳回类型（必选，单选）、驳回原因（必填，多行文本）
7. 未来规划：无额外必填字段
8. 评审意见：多行文本，非必填
9. 取消：关闭弹窗，不保存
10. 提交：校验必填项 → 保存到 `requirement_review` 表 → 关闭弹窗 → 刷新列表

- 筛选条件显示在列表上方，支持单个清除和全部清除

| 创建人 | 多选下拉 | 来自CSV数据 |
| 创建时间范围 | 日期区间选择器 | 开始日期 ~ 结束日期 |
| 最后修改时间范围 | 日期区间选择器 | 开始日期 ~ 结束日期 |
| 优先级 | 多选下拉 | 高 / 中 / 低 / 未标注 |
| 评审分类 | 多选下拉 | 通过 / 驳回 / 未来规划 |
| 项目 | 多选下拉 | 已关联的项目列表 |

**列表字段**

| 需求名称 | ✓ | 来自CSV |

| 部门 | ✓ | 来自CSV |
| 创建人 | ✓ | 来自CSV |

### 5.4 核心功能实现

**CSV 导入与版本对比**

1. 解析上传的 CSV 文件
2. 对每条记录，计算 `version_hash`（对 `demand_desc + businessobjective + department + importance + reason_type` 取 MD5）
3. 查询 `requirements` 表中是否存在 `demand_id`：
   - 不存在 → 插入新记录，`is_reviewed = 0`
   - 存在但 hash 不同 → 更新记录，`is_updated = 1`，`is_reviewed = 0`
   - 存在且 hash 相同 → 保持现状
4. 同步更新 `requirement_review` 表记录
5. 返回导入结果统计

**积压超时预警计算**

```javascript
// 计算超期天数（排除周末）
function calculateOverdueDays(createdDate, targetDays = 20) {
  const created = new Date(createdDate);
  const today = new Date();
  let workDays = 0;
  let current = new Date(created);
  
  while (current <= today) {
    const dayOfWeek = current.getDay();
    if (dayOfWeek !== 0 && dayOfWeek !== 6) {
      workDays++;
    }
    current.setDate(current.getDate() + 1);
  }
  
  return Math.max(0, workDays - targetDays);
}
```

**项目归属逻辑**

- 评审通过时必须选择项目归属
- 选择"新建项目"：创建 `projects` 记录，生成 `project_id`，关联到 `requirement_review.project_id`
- 选择"归属现有项目"：直接关联 `projects.project_id`

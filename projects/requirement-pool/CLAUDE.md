# 需求池管理系统 MVP

## 项目概述

轻量级需求池管理工具，支持评审流程、驳回类型、版本对比、导出和 AI 总结，以及未整合需求跟踪、已整合快到期预警、AI 需求质量分析。

## 快速命令

```bash
cd projects/requirement-pool/

# 启动后端（开发模式）
python -m uvicorn app.main:app --reload --port 5173

# 初始化数据库（首次运行）
python scripts/init_db.py

# 导入 CSV 数据
python scripts/import_csv.py --file ../../data-analysis-local/需求池评估报告-20260514/raw/需求池需求.csv
```

## 架构

| 模块 | 文件 | 职责 |
|------|------|------|
| 后端 API | `app/main.py` | FastAPI 应用入口 |
| 数据模型 | `app/models.py` | Pydantic + SQLAlchemy 模型 |
| 路由 | `app/routers/` | 需求 CRUD、状态流转、评审、看板 API |
| 前端 | `app/static/index.html` | 单页应用 |
| 脚本 | `scripts/` | 数据库初始化、CSV 导入 |
| AI 服务 | `app/services/ai.py` | 本地大模型调用（质量分析 + 评审摘要） |

## 技术栈

- **后端**：Python 3.10+ / FastAPI / SQLAlchemy / SQLite
- **前端**：HTML5 + Vanilla JavaScript（无框架）
- **图表**：ECharts（CDN）
- **Excel 导出**：openpyxl
- **AI**：支持配置本地大模型（兼容 OpenAI API 格式）

## MVP 功能

| 功能 | 描述 |
|------|------|
| F1 需求录入 | CSV 批量导入 + 单条增删改查 + 版本对比 + 未评审标记 |
| F2 评审标注 | 优先级（高/中/低）+ 评审确定上线时间 + 评审备注 |
| F3 驳回类型 | 8 种标准化驳回类型 + 原因描述 |
| F4 状态流转 | 待处理 → 处理中 → 已整合 → 已关闭（internal_status 工作流） |
| F5 过期预警 | 自动标记 + 首页告警 + 批量处理入口 |
| F6 积压看板 | 关键指标 + 多维分布 + 积压清单 |
| F7 评审导出 | Excel 导出，支持手工录入外部系统 |
| F8 评审看板 | 当月评审统计 + 效率指标 + 驳回分布 |
| F9 AI 总结 | 本地大模型生成评审摘要，支持推送 |
| F10 未整合需求跟踪 | pending/processing 持续显示，新增未整合高亮，积压30天标红 |
| F11 已整合快到期预警 | 距期望上线 ≤30 天预警，已过期单独告警 |
| F12 AI 需求质量分析 | 评分 0-100 + 改进建议，辅助识别低质量需求 |

## 数据来源

兼容现有 CSV 格式（字段：编号、所属阶段、归属项目、干系部门、需求名称、状态、期望上线日期、业务现状、需求描述、期望达成的业务效果、由谁创建、创建时间、需求来源部门、业务能力、业务重要程度）

## 状态设计

| 字段 | 来源 | 说明 |
|------|------|------|
| source_status | CSV 导入，只读 | 原始状态（已关闭/待处理/处理中/已整合），用于判断 `is_closed` |
| internal_status | 系统内部 | 工作流状态：pending/processing/integrated/closed |
| is_closed | 派生 | `source_status="已关闭"` 时为 1，用于评审筛选 |

**注意**：`is_closed` 来自 CSV 原始数据，不参与状态流转。评审时只处理 `is_closed=0` 的活跃需求。

## 评审驳回类型

| 类型 | 编码 | 原因描述 |
|------|------|---------|
| 优先级过高 | P001 | 需求紧急程度与业务价值不匹配 |
| 重复需求 | P002 | 与已有需求重复，建议合并 |
| 范围不清 | P003 | 需求描述模糊，无法评估 |
| 依赖缺失 | P004 | 存在未识别的上游依赖 |
| 资源不足 | P005 | 无足够人力/技术资源承接 |
| 业务调整 | P006 | 业务方向调整，暂不纳入 |
| 条件不成熟 | P007 | 前提条件不满足 |
| 其他 | P999 | 其他原因，需在备注中说明 |

## AI 质量评分维度

| 维度 | 权重 | 评分规则 |
|------|------|---------|
| 描述完整性 | 40% | 描述 ≥50 字符满分，<20 字符 0 分 |
| 效果清晰度 | 30% | 效果描述 ≥20 字符满分，<10 字符 0 分 |
| 依赖识别 | 15% | 是否提及依赖关系 |
| 范围明确性 | 15% | 是否包含具体范围（非模糊词） |

## 已知问题

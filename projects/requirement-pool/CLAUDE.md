# 需求池管理系统 MVP

## 项目概述

轻量级需求池管理工具，支持评审流程、驳回类型、版本对比、导出和 AI 总结，以及未关联需求跟踪、超期未评审预警、AI 需求质量分析。

## 快速命令

```bash
cd projects/requirement-pool/

# 启动后端（开发模式）
python -m uvicorn app.main:app --reload --port 5173

# 初始化数据库（首次运行）
python scripts/init_db.py

# 导入 CSV 数据
python scripts/import_csv.py --file ../../data-analysis-local/需求池评估报告-20260514/raw/需求池需求.csv
```

## 架构

| 模块 | 文件 | 职责 |
|------|------|------|
| 后端 API | `app/main.py` | FastAPI 应用入口 |
| 数据模型 | `app/models.py` | Pydantic + SQLAlchemy 模型 |
| 路由 | `app/routers/` | 需求 CRUD、状态流转、评审、看板 API |
| 前端 | `app/static/index.html` | 单页应用 |
| 脚本 | `scripts/` | 数据库初始化、CSV 导入 |
| AI 服务 | `app/services/ai.py` | 本地大模型调用（质量分析 + 评审摘要） |

## 技术栈

- **后端**：Python 3.10+ / FastAPI / SQLAlchemy / SQLite
- **前端**：HTML5 + Vanilla JavaScript（无框架）
- **图表**：ECharts（CDN）
- **Excel 导出**：openpyxl
- **AI**：支持配置本地大模型（兼容 OpenAI API 格式）

## 数据来源

兼容现有 CSV 格式（字段：编号、所属阶段、归属项目、干系部门、需求名称、状态、期望上线日期、业务现状、需求描述、期望达成的业务效果、由谁创建、创建时间、需求来源部门、业务能力、业务重要程度）

## 状态设计

| 字段 | 来源 | 说明 |
|------|------|------|
| demand_status | CSV 导入，只读 | 原始状态：active / closed |
| plan_status | 系统内部 | 规划状态：正常 / 未来规划 |
| is_closed | 派生 | `demand_status="closed"` 时为 1，用于评审筛选 |

**积压超时触发条件**：demand_status=active + business_id IS NULL + plan_status != '未来规划' + 最后修改时间距今 > 10 个工作日

## 评审驳回类型

| 类型 | 编码 | 原因描述 |
|------|------|---------|
| 优先级过高 | P001 | 需求紧急程度与业务价值不匹配，建议降低优先级排队 |
| 重复需求 | P002 | 与已有需求重复，功能重叠，建议合并或引用已有需求 |
| 范围不清 | P003 | 需求描述模糊，无法准确评估范围和工作量，需补充说明 |
| 依赖缺失 | P004 | 存在未识别的上游依赖或关联系统，需先解决依赖问题 |
| 资源不足 | P005 | 当前无足够人力或技术资源承接，建议调整时间或拆解 |
| 业务调整 | P006 | 业务方向调整或战略变更，暂不纳入本期范围 |
| 条件不成熟 | P007 | 前提条件不满足（如需先完成XX系统改造），建议暂缓 |
| 其他 | P999 | 其他原因，需在备注中说明 |

## AI 质量评分维度

| 维度 | 权重 | 评分规则 |
|------|------|---------|
| 描述完整性 | 40% | demand_desc ≥50 字符满分，<20 字符 0 分 |
| 效果清晰度 | 30% | businessobjective ≥20 字符满分，<10 字符 0 分 |
| 依赖识别 | 15% | 是否提及依赖关系 |
| 范围明确性 | 15% | 是否包含具体范围（非模糊词） |

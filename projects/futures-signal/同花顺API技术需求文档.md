## 一、项目背景与目标

### 1.1 项目概述

本团队计划基于期货持仓数据构建一套**自动化交易信号系统**，用于生成多空交易信号，辅助投资决策。

### 1.2 核心目标

通过同花顺 API 接口获取以下数据维度：
- 成交量、持仓量
- 主力合约价格（收盘价）
- 主要期货品种的特定席位持仓变化（如棕榈油的摩根大通席位）

### 1.3 技术能力说明

本团队技术背景为**非编程人员**，希望同花顺方面提供：
- 封装好的 Python 脚本模板
- 账号密码配置后即可运行
- 面向技术小白的教程和技术支持

## 二、交易策略逻辑

### 2.1 多仓信号逻辑（做多）

| 条件 | 描述 | 数据来源 |
|------|------|----------|
| 条件1 | 成交量增加（当日成交量 > 前一日成交量） | 行情API |
| 条件2 | 持仓量增加（当日持仓量 > 前一日持仓量） | 行情API |
| 条件3 | 价格上涨（当日收盘价 > 前一日收盘价） | 行情API |
| 条件4 | 目标席位（如摩根大通）多头持仓增加 > 2000手 | 会员持仓API |

**触发信号**：条件1 + 条件2 + 条件3 + 条件4 同时满足 → **做多**

| 条件3 | 价格回落（当日收盘价 < 前一日收盘价） | 行情API |

## 三、数据需求清单

### 3.1 基础行情数据

| 字段名 | 说明 | 示例值 |
|--------|------|--------|
| 合约代码 | 期货主力连续合约 | P.MAIN 或 P2609.DCE |
| 收盘价 | 当前最新成交价 | 8250 元/吨 |
| 成交量 | 当日累计成交量 | 125,000 手 |
| 持仓量 | 当日收盘持仓量 | 350,000 手 |
| 涨跌幅 | 当日价格变动百分比 | +1.25% |

### 3.2 会员持仓数据

| 字段名 | 说明 | 示例值 |
|--------|------|--------|
| 会员名称 | 期货公司席位名称 | 摩根大通 |
| 多头持仓 | 该席位多头持仓量 | 25,000 手 |
| 空头持仓 | 该席位空头持仓量 | 8,000 手 |
| 持仓变化 | 较上一交易日增减量 | +2,500 手 |

## 四、API 功能需求

### 4.1 认证功能

- 账号密码登录认证
- 支持记住登录状态

### 4.2 数据获取功能

- 获取期货主力连续合约行情数据
- 获取指定席位的会员持仓数据
- 支持按品种、席位号筛选

### 4.3 数据导出功能

- 支持导出为 CSV 或 Excel 格式
- 包含完整字段信息

### 4.4 偏好调用方式

**首选**：Python SDK 方式（封装脚本模板）

**备选**：Excel 插件方式（手动导出数据）

## 五、数据规格说明

### 5.1 数据频率要求

| 数据类型 | 频率 | 说明 |
|----------|------|------|
| 日线数据 | 每日收盘后更新 | 用于计算持仓量变化和席位持仓变化 |
| 会员持仓 | 每日收盘后更新 | 交易所每日公布会员持仓数据 |

### 5.2 监控规模

- 监控合约：10 个（农产品主力连续合约）
- 关注席位：2-3 个（如摩根大通期货席位）

### 5.3 数据更新时效

请明确以下信息：
- 会员持仓数据的数据更新时间（UTC+8）
- 数据是否存在延迟及延迟时长

### 5.4 API 调用限制

请提供以下信息：
- 日调用频率限制
- 单次请求数据量限制
- 并发连接数限制
- 超出限制的处理方式

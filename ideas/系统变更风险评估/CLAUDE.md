# CLAUDE.md — 系统变更风险评估

> 系统变更风险评估数据分析工作目录，用于IT系统变更的风险评估、审批流程追踪、变更记录分析等。
> 数据来源包括风险评估表、审批记录、变更实施记录等。

---

## 数据源

| 数据源 | 路径 | 说明 |
|--------|------|------|
| 风险评估表 | `raw/` | 系统变更风险评估表（Excel/表单） |
| 审批记录 | `raw/` | 变更审批签字记录 |
| 实施记录 | `raw/` | 变更实施Checklist记录 |
| 分析结果 | `metadata/` | 评估结果汇总、分析报告 |

---

## 数据字典

完整数据字典位于：
- `metadata/` — 风险评估维度说明、评分标准、指标定义

包含：
- `风险评估维度.md` — 直接风险、衍生风险、流程变更、权限变更、数据变更、合规风险
- `评分标准.md` — 影响程度/发生概率评分矩阵、风险等级判定
- `审批权限矩阵.md` — 各风险等级对应的审批层级

---

## 分析类型

| 分析类型 | 说明 |
|----------|------|
| 变更记录分析 | 按时间维度统计变更数量、类型分布 |
| 风险等级分析 | 各风险等级占比、高风险变更趋势 |
| 审批时效分析 | 审批周期分布、时效达标率 |
| 变更类型分析 | 按系统、按类型、按部门的变更分布 |
| 回滚分析 | 回滚触发率、回滚原因分析 |

---

## 核心指标

| 指标 | 说明 |
|------|------|
| 风险分值 | 影响程度 × 发生概率（1-25分） |
| 风险等级 | 低(1-4)/中(5-9)/高(10-14)/极高(15-25) |
| 审批时效 | 从提交到完成审批的耗时 |
| 回滚率 | 触发回滚的变更占比 |
| 按时完成率 | 在变更窗口内完成的变更占比 |

## 目录结构

```
系统变更风险评估/
├── CLAUDE.md                         # 本文件
├── metadata/                         # 数据字典和元数据
│   ├── 风险评估维度.md
│   ├── 评分标准.md
│   └── 审批权限矩阵.md
├── raw/                              # 原始数据文件（Excel/CSV）
├── scripts/                          # 分析脚本模板
│   └── analysis_template.py
└── <YYYYMMDD>/                      # 每次分析任务子目录
    ├── README.md                     # 分析说明
    ├── analyze_*.py                  # 分析脚本
    ├── data.json                     # 分析数据
    ├── index.html                    # HTML报告
    └── charts/                       # 图表输出
```

**命名规范**：`系统变更风险评估/<YYYYMMDD>/`，如 `20260519变更分析/`

---

## 分析流程

1. **数据确认** — 检查 `raw/` 目录中数据文件完整性
2. **创建任务目录** — 在 `系统变更风险评估/<YYYYMMDD>/` 下创建本次分析目录
3. **编写脚本** — 在本次任务目录下创建分析脚本
4. **执行分析** — `python3 <YYYYMMDD>/analyze_*.py`
5. **生成报告** — 输出 HTML 报告到 `系统变更风险评估/<YYYYMMDD>/`

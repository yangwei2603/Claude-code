# 风险评估维度说明

## 六大评估维度

| 维度 | 说明 | 评估项 |
|------|------|--------|
| 直接风险 | 变更对系统本身的直接影响 | 系统可用性、功能正确性、数据完整性、性能影响、稳定性 |
| 衍生风险 | 由直接风险衍生出来的间接影响 | 上下游影响、业务连续性、回滚风险、第三方依赖 |
| 流程变更风险 | 对业务流程和审批流的影响 | 业务流程影响、审批流变化、监控/告警 |
| 权限变更风险 | 对账号权限和角色管理的影响 | 账号权限变更、角色变更、特权账号管理、离职/转岗交接 |
| 数据变更风险 | 对数据结构和数据质量的影响 | 数据结构变更、敏感数据处理、数据质量、数据备份 |
| 合规风险 | 对监管合规和审计追溯的影响 | 监管合规、行业规范、内部制度、审计追溯 |

## 评分标准

- 影响程度：1-5分（1=极低，5=极高）
- 发生概率：1-5分（1=极低，5=极高）
- 风险分值 = 影响程度 × 发生概率

| 风险分值 | 风险等级 | 颜色标识 |
|----------|----------|----------|
| 1-4 | 低风险 | 绿 |
| 5-9 | 中风险 | 黄 |
| 10-14 | 高风险 | 橙 |
| 15-25 | 极高风险 | 红 |

**最终风险等级取所有维度最高分**
